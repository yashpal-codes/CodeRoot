# HTML Form

## HTML `<form >` tag:

- This Form element is a container for all input elements in HTML.
- This is the powerhouse for all user input related works in a webpage.

<FontStyle text="# Form attributes" heading="h4" color="blue" />

<FontStyle text=" 1. Action" heading="h5" color="orange" />

- Action attribute defines
  the endpoint or url where data to be sent when use click on submit form button.
- Value: The url ( absolute or relative )

<FontStyle text=" 2. Target " heading="h5" color="orange" />

- Specifies where to display the response after form is submitted.

- \_self : default, current window/tab.
- \_parent : Parent frame.
- \_blank : New window/tab.
- \_top : The top level frame.

<FontStyle text=" 3. Method " heading="h5" color="orange" />

- Defines the HTTP method to be used to send data to server.
- Value : get/post/put/delete

<FontStyle text=" 4. Autocomplete " heading="h5" color="orange" />

- Defines autocomplete should be on/off when user is using the form in webpage.

## HTML Form input elements:

- HTML Form have many input elements, we will discuss here some of mostly used input elements.

1. HTML input tag.
2. HTML label tag.
3. HTML select tag.
4. HTML fieldset & legend

<FontStyle text=" 1. Input Tag " heading="h4" color="blue" />

- Input element is used to get inputs from user, this element renders a input field in browser where user
  can fill their details.

- Input have many attributes, combining them we we can render a field to get required user information.

- Input is a void tag.

<FontStyle text=" 1. Type " heading="h5" color="orange" />

- Define type of input, like button, text, radio, check-box, password, range, search and date.

<FontStyle text=" 2. Id " heading="h5" color="orange" />

- Id attribute, this is used to bind label and input tags together, label 'for' tag and input 'id' tag must be same to bind them together.

- By keeping both same label and input tags are bind, so label is describing the input tag.
- This is very useful for screen reader feature in web page.

<FontStyle text=" 3. Name " heading="h5" color="orange" />

- Input name is necessary as it defines the description of the input filed and it is required when input result is submitted at server.

- When a `<form>` is submitted, the browser collects all the inputs that have a name attribute and sends their name=value pairs to the server. This happens via an HTTP request (usually GET or POST, depending on the form’s method attribute).

- It acts like a key in a key-value pair, where the name is the key and the input’s value (what the user enters or selects) is the value. This attribute is critical for form processing—without it, the input’s data won’t be sent to the server or included in the form’s data payload.

```HTML
<form action="/submit" method="POST">
  <input type="text" name="username" value="JohnDoe" />
  <input type="submit" value="Submit" />
</form>
```

- When submitted, the server receives something like username=JohnDoe in the request body (for POST) or in the URL query string (for GET, e.g., /submit?username=JohnDoe). If the name attribute were omitted, the input’s data wouldn’t be included in the submission.

<FontStyle text=" 4. Readonly " heading="h5" color="orange" />

- This attribute makes input element read only, means user cannot change the field data.

<FontStyle text=" 5. Disable " heading="h5" color="orange" />

- This is same as a readonly, but additionally make input element can't select and disable copy and paste.

<FontStyle text=" 6. Placeholder " heading="h5" color="orange" />

- Placeholder attribute set placeholder content for input element.

<FontStyle text=" 7. Value " heading="h5" color="orange" />

- Value attribute sets the initial value for input element.

<FontStyle text=" 8. Required " heading="h5" color="orange" />

- If required attribute is added, means it is compulsory to fill this input field.

<FontStyle text=" Others are Size, Min and Max " heading="h5" color="orange" />

function FormExample() {

  const handleSubmit = (e) => {
    e.preventDefault();
    alert("Form submitted! Check the console for data.");

  };

return (

<form onSubmit={handleSubmit} >

  />

<label htmlFor="textInput"> Text Input: </label>

  id="searchInput"
  name="search"

<input type="submit" value="Submit Form" />

</form>

);
